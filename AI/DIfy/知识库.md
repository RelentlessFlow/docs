# DIfy 知识库

## 分段设置

### 通用

**分段标识符**

一般默认

**分段最大长度、分段重叠长度**

| **数据类型**                 | **推荐分块大小** | **推荐重叠长度**  | **适用场景**               |
| ---------------------------- | ---------------- | ----------------- | -------------------------- |
| **普通文本（政策、论文）**   | 500 tokens       | **50 tokens**     | **默认推荐**，信息均匀     |
| **技术文档、代码**           | 700 tokens       | **100 tokens**    | 代码上下文需要更长         |
| **长篇对话、访谈**           | 400 tokens       | **75-100 tokens** | 避免语境丢失               |
| **FAQ 知识库**               | Q&A 分段         | 0                 | **无须重叠**，问答是独立的 |
| **结构化数据（表格、列表）** | 300-500 tokens   | **50 tokens**     | 避免表格被拆散             |

**文本预处理规则**

替换掉连续的空格、换行符和制表符，默认勾选

删除所有 URL 和电子邮件地址，看场景，客服这种，留着吧

**使用 Q&A 分段**

Q&A 它是一种特殊的数据

```
Q： 什么是金融风险控制？
A： 金融风险控制指的是……
```

Q&A 适用场景：

| **文档类型**           | **是否用 Q&A 分段？** | **推荐配置**                     |
| ---------------------- | --------------------- | -------------------------------- |
| FAQ / 问答手册         | ✅ 适合                | **Q&A 分段**，每条问答一块       |
| 产品使用手册           | ✅ 适合                | **Q&A 分段**，按功能模块拆分     |
| 法律法规（含具体问答） | ✅ 适合                | **Q&A 分段**，确保法规条款可问答 |
| **公司介绍**           | ❌ 不适合              | **普通分段**，300-500 tokens     |
| **科研论文**           | ❌ 不适合              | **普通分段**，500-800 tokens     |
| **政策法规（非问答）** | ❌ 不适合              | **普通分段**，500-800 tokens     |

### 父子分段

**父块用作上下文**

**段落：**标识符默认就行，分段最大长度，文献就700-1000

**全文：**适合小型文档，长文档计算消耗比较大

📌 适合用“全文”的情况**

✅ **单篇长度适中（少于 10,000 tokens）\**的政府报告
✅ 需要\**全文语境**的场景，比如法律条文、合同类文件
✅ **FAQ 结构**，因为回答通常依赖整个文档

不推荐全文的情况**

❌ **长文档（超过 10,000 tokens）**，因为处理成本高，AI 可能丢失关键细节
❌ **分层结构清晰的政策法规**，父子分段的“段落”模式更适合

**子块用于检索**

全默认就行，其他参考通用分段设置

### 索引方式

高质量就是RAG、经济就是构建一个关键字字典，Top K就是返回几项，太大了影响性能

Embedding 模型，用百炼的V3就完事了

### 检索设置

向量检索就是做文本相似度，相当于 `LIKE`，全文检索就是 `=`，混合就是两个都用。

用混合 + 全默认参数就完事了。

**Score 阈值：**

通过设置一个最低分数（阈值），可以确保只有那些被认为足够相关的文档或信息被返回给用户。低于这个阈值的结果将被自动排除。（质量控制、消除噪音等）

1. 数据干净，可以不设置阈值。
2. 阈值高低取决于用户是否愿意看到更多相关信息。

通过合理设置Score阈值和其他参数，可以在召回率和精确率之间找到一个合适的平衡点，以满足特定应用的需求。
